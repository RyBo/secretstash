How it works

  
This secret was split into shares using Shamir's Secret Sharing. A set number of shares (the quorum) is needed to unlock it; fewer than that reveal nothing.

  
Combining happens entirely in your browser. The shares are turned back into the unlock token locally, and only that token, never the shares, is sent to the server to retrieve the secret. The server only keeps a hashed form of the token, so it cannot read the secret on its own.

  
The secret works for the number of reads the sender picked, then it is deleted for good. Secrets are kept in memory only; restarting the server erases all of them.

  
Built in Go. The full source is on GitHub if you want to check it yourself.
