How it works

  
secretstash makes a one-time link for a secret. The link works for the number of reads the sender picked, then the secret is deleted for good. It also expires after a time the sender set.

  
The token that unlocks the secret lives in the part of the link after the #. Browsers never send that part to the server, so it stays out of server logs and link previews. The server only keeps a hashed form of the token, so it cannot read the secret or rebuild a link once it has been shown.

  
Secrets are kept in memory only. Restarting the server erases all of them.

  
Built in Go. The full source is on GitHub if you want to check it yourself.
